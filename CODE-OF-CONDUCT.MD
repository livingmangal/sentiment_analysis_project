# Code of Conduct

## Our Pledge
We, as contributors and maintainers of the **Sentiment Analysis Web Application**, pledge to create a respectful, inclusive, and harassment-free environment for everyone involved in this project.

We value collaboration, open discussion, and constructive feedback, and we aim to make participation in this project a positive experience for all.

---

## Our Standards

### Expected behavior
Participants are expected to:
- Use respectful and inclusive language
- Be considerate of different viewpoints and experiences
- Provide constructive and actionable feedback
- Accept criticism gracefully and professionally
- Focus on improving the project and helping the community

### Unacceptable behavior
Unacceptable behavior includes, but is not limited to:
- Harassment, discrimination, or personal attacks
- Trolling, insulting, or deliberately disruptive comments
- Publishing private or sensitive information without consent
- Any form of abusive, threatening, or unprofessional conduct

---

## Maintainer Responsibilities
Project maintainers are responsible for:
- Enforcing this Code of Conduct fairly and consistently
- Reviewing issues, pull requests, and discussions respectfully
- Taking appropriate action when unacceptable behavior occurs

Maintainers have the right to remove, edit, or reject contributions that violate this Code of Conduct.

---

## Scope
This Code of Conduct applies to:
- All project spaces, including GitHub issues, pull requests, discussions, and repositories
- All interactions related to this project, whether online or offline

---

## Enforcement
If you experience or witness behavior that violates this Code of Conduct, please report it to the project maintainers.

All reports will be handled confidentially and reviewed promptly.  
Actions may include warnings, temporary restrictions, or permanent removal from the project, depending on severity.
